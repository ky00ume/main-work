# 문서 업데이트 요청 — 비전 타운 RPG 봇

## 목적
BOT 코드 변경 후 관련 문서를 최신 상태로 유지합니다.

## 프로젝트 문서 구조
- `README.md` — 봇 기본 설명
- `PATCH_NOTES_DEV.md` — 개발자용 패치 노트 (변경 상세 내역)
- `PATCH_NOTES_USER.md` — 유저용 패치 노트 (추가된 기능 안내)
- `UX_개선_PLAN.md` — UX 개선 계획

## 문서 분석 절차

1. **변경 사항 파악**
   - 추가된 기능 목록
   - 수정된 기능 목록
   - 제거된 기능 목록
   - 버그 수정 내역

2. **영향 받는 문서 결정**
   - 유저가 알아야 할 변경 → `PATCH_NOTES_USER.md`
   - 개발 상세 내역 → `PATCH_NOTES_DEV.md`
   - 봇 사용법 변경 → `README.md`

## 문서 업데이트 형식

### PATCH_NOTES_DEV.md 형식
```markdown
## v[버전] — [날짜]
### 추가
- [파일명] [함수/기능명]: [변경 내용]

### 수정
- [파일명]: [수정 내용]

### 버그 수정
- [증상]: [원인 및 수정 내용]
```

### PATCH_NOTES_USER.md 형식
```markdown
## [날짜] 업데이트
### ✨ 새로운 기능
- [커맨드명]: [기능 설명]

### 🔧 개선
- [기능명]: [개선 내용]

### 🐛 버그 수정
- [증상] 수정
```

## 업데이트 가이드라인
- 기술 용어는 유저용 노트에서 쉬운 말로 바꾸기
- 커맨드 변경 시 예시 포함
- 기존 내용은 유지하고 최신 내용을 맨 위에 추가
